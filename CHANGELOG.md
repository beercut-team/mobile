## [1.0.0] - 2026-02-26

### ✨ Новые возможности

- **Система уведомлений**: Полноценная система push-уведомлений с 10 типами событий
  - Изменение статуса пациента
  - Назначение врача/хирурга
  - Планирование операции
  - Установка диагноза и типа операции
  - Новые комментарии
  - Обновления чек-листов
  - Расчеты ИОЛ
  - Загрузка медиафайлов
  - Счетчик непрочитанных уведомлений
  - Навигация к пациенту при нажатии на уведомление

- **Управление пациентами**: Полный цикл работы с пациентами
  - Создание и редактирование карточек пациентов
  - Ролевой доступ (районный врач, хирург, пациент, администратор)
  - Workflow статусов: НОВЫЙ → ПОДГОТОВКА → ТРЕБУЕТ ПРОВЕРКИ → ОДОБРЕН → ОПЕРАЦИЯ ЗАПЛАНИРОВАНА → ЗАВЕРШЕН/ОТКЛОНЕН
  - Паспортные данные и медицинская информация
  - Доступ пациентов по коду доступа

- **Калькулятор ИОЛ**: Расчет силы интраокулярной линзы
  - Поддержка формул: SRKT, Haigis, Hoffer Q
  - История расчетов для каждого пациента
  - Предупреждения о значениях вне диапазона

- **Система чек-листов**: Предоперационные чек-листы
  - Отслеживание статуса (в ожидании, в процессе, завершен, отклонен, истек)
  - Обязательные и опциональные пункты
  - Расчет прогресса выполнения

- **Комментарии**: Система комментариев с треддингом
  - Флаг срочности
  - Статус прочитанности
  - Коммуникация врач-пациент

- **Медиафайлы**: Загрузка и управление файлами
  - Изображения (через expo-image-picker)
  - Документы (через expo-document-picker)
  - Миниатюры и категории
  - Подписанные URL для скачивания

- **Печать/PDF**: Генерация PDF-документов
  - Маршрутные листы
  - Отчеты по чек-листам
  - Платформо-специфичная загрузка (web/mobile)

- **Медицинские стандарты**: Поддержка международных стандартов
  - HL7/FHIR R4 (Patient, Condition, Procedure, Observation)
  - ICD-10 (коды диагнозов)
  - SNOMED CT (коды процедур)
  - LOINC (наблюдения)
  - FHIR-маппер для конвертации данных пациентов

- **Интеграции**: Заглушки для внешних систем
  - ЕМИАС (Москва)
  - РИАМС (10 регионов: Москва, Санкт-Петербург и др.)
  - Валидация и экспорт данных
  - Отслеживание статуса синхронизации

### 🔧 Улучшения

- **Офлайн-режим**: Полная поддержка работы без интернета
  - Очередь мутаций в AsyncStorage
  - Автоматическая синхронизация при восстановлении соединения
  - Индикатор офлайн-статуса
  - Мониторинг сети через @react-native-community/netinfo

- **Аутентификация**: JWT-токены с автоматическим обновлением
  - Refresh tokens
  - Безопасное хранение (SecureStore на iOS/Android, localStorage на web)
  - Автоматический refresh при 401
  - Защита от race conditions

- **UI/UX**: Современный интерфейс с анимациями
  - Светлая/темная тема
  - Режим высокой контрастности для слабовидящих
  - Увеличение шрифтов (2.0x) в режиме accessibility
  - Плавающая панель табов с blur-эффектом
  - Toast-уведомления с Reanimated анимациями
  - TabView для многосекционных экранов
  - ActionSheet для контекстных меню

- **Производительность**: Оптимизация запросов и кеширования
  - React Query с 5-минутным stale time
  - Автоматическая инвалидация после синхронизации
  - Оптимистичные обновления с откатом при ошибках

### 🛠️ Технические улучшения

- **Архитектура**: Чистая архитектура с разделением слоев
  - Централизованный API-клиент в `/lib/api.ts`
  - Доменные модули (auth, patients, notifications, districts, surgeries, checklists, comments, iol, media, print, sync)
  - Custom hooks для инкапсуляции логики
  - Provider-иерархия: QueryClient → Accessibility → Auth → Theme → Toast

- **Роутинг**: Expo Router с file-based routing
  - Две группы маршрутов: `/(auth)` и `/(tabs)`
  - Защищенные маршруты с автоматическим редиректом
  - Typed routes для type safety
  - Скрытие табов по ролям через `href: null`

- **Тестирование**: Standalone Node.js тесты
  - Тесты медицинских стандартов (ICD-10, SNOMED, LOINC, FHIR)
  - Тесты интеграций (EMIAS, RIAMS)
  - Скрипты для тестирования уведомлений
  - Запуск всех тестов через `npm run test`

- **Seeding**: Скрипты для генерации тестовых данных
  - `npm run seed` - все данные (логин, 5 пациентов, комментарии, ИОЛ)
  - `npm run seed:patients` - только пациенты
  - `npm run seed:comments` - комментарии
  - `npm run seed:iol` - расчеты ИОЛ
  - `npm run seed:statuses` - обновление статусов
  - `npm run seed:medical` - медицинские метаданные
