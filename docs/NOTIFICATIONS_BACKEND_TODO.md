# TODO: Реализация системы уведомлений на бэкенде

## Проблема

Пациент Айаал Степанов (ID: 2, код доступа: `e5f6g7h8`) имеет полные медицинские данные:
- Статус: APPROVED
- Диагноз: Открытоугольная глаукома II стадии
- Операция: ANTIGLAUCOMA (OS - левый глаз)
- Дата операции: 2026-11-11
- Врач: Николаев Айсен
- Хирург: Васильев Ньургун

Однако у пациента **0 уведомлений** в системе. При изменении данных пациента (статус, назначение врача, дата операции и т.д.) пациенты должны получать уведомления об этих изменениях.

## Текущее состояние

### Что работает ✅
- Фронтенд может читать уведомления: `GET /api/v1/notifications`
- Фронтенд может отмечать как прочитанное: `POST /api/v1/notifications/{id}/read`
- UI для уведомлений полностью реализован
- Push-уведомления настроены через Expo

### Что НЕ работает ❌
- **API endpoint не существует:** `POST /api/v1/notifications` возвращает 404
- **Бэкенд не создает уведомления автоматически** при изменении данных пациента
- Фронтенд не может создавать уведомления (это должен делать бэкенд)

## Что нужно реализовать

### 1. API Endpoint для создания уведомлений

```
POST /api/v1/notifications
Authorization: Bearer <token>

Request Body:
{
  "user_id": 2,
  "patient_id": 2,
  "title": "Статус изменен",
  "message": "Ваш статус изменен на: На подготовке",
  "type": "status_change"
}

Response: 201 Created
{
  "success": true,
  "data": {
    "id": 123,
    "user_id": 2,
    "patient_id": 2,
    "title": "Статус изменен",
    "message": "Ваш статус изменен на: На подготовке",
    "type": "status_change",
    "is_read": false,
    "created_at": "2026-02-26T10:30:00Z"
  }
}
```

### 2. Автоматическое создание уведомлений

При изменении данных пациента через `PATCH /api/v1/patients/{id}` автоматически создавать уведомления:

| Изменяемое поле | Тип уведомления | Пример сообщения |
|-----------------|-----------------|------------------|
| `status` | `status_change` | "Ваш статус изменен на: На подготовке" |
| `doctor_id` | `doctor_assigned` | "Вам назначен лечащий врач: Николаев Айсен" |
| `surgeon_id` | `surgeon_assigned` | "Вам назначен хирург: Васильев Ньургун" |
| `surgery_date` | `surgery_scheduled` | "Ваша операция назначена на 11 ноября 2026" |
| `diagnosis` | `diagnosis_set` | "Установлен диагноз: Открытоугольная глаукома" |
| `operation_type` | `operation_type_set` | "Назначена операция: Антиглаукоматозная (левый глаз)" |

Также создавать уведомления при:
- Добавлении комментария: `POST /api/v1/comments`
- Обновлении чек-листа: `PATCH /api/v1/checklists/{id}/items/{item_id}`
- Расчете ИОЛ: `POST /api/v1/iol-calculations`
- Загрузке документа: `POST /api/v1/media`

### 3. Интеграция с Push-уведомлениями

При создании уведомления в БД также отправлять push-уведомление через Expo Push Notification Service:

```json
{
  "to": "ExponentPushToken[xxxxxxxxxxxxxxxxxxxxxx]",
  "sound": "default",
  "title": "Статус изменен",
  "body": "Ваш статус изменен на: На подготовке",
  "data": {
    "patient_id": 2,
    "type": "status_change"
  },
  "badge": 1
}
```
