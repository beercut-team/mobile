## Категории исправлений

### 🔴 Критический приоритет (5 проблем)

#### 1. Офлайн-синхронизация: потеря данных
**Проблема:** Мутации терялись при восстановлении соединения из-за отсутствия персистентности очереди.

**Решение:**
- Реализована персистентная очередь в AsyncStorage (`/lib/offline-queue.ts`)
- Добавлены timestamps для отслеживания порядка операций
- Автоматическая синхронизация при восстановлении сети через `@react-native-community/netinfo`
- Hook `useOfflineSync` для управления состоянием синхронизации

**Impact:** Критический. Предотвращена потеря пользовательских данных в офлайн-режиме.

---

#### 2. XSS-уязвимость в комментариях
**Проблема:** Отсутствие санитизации HTML в пользовательском контенте.

**Решение:**
- Интегрирован DOMPurify для web-платформы
- Добавлена санитизация в `CommentThread` компоненте
- Платформо-специфичная обработка (web: DOMPurify, mobile: React Native Text)

**Impact:** Критический. Закрыта серьезная уязвимость безопасности.

---

#### 3. Небезопасное хранение токенов
**Проблема:** Токены хранились в незащищенном AsyncStorage на всех платформах.

**Решение:**
- Создан модуль `/lib/token-storage.ts` с платформо-специфичным хранением
- iOS/Android: SecureStore (аппаратное шифрование)
- Web: localStorage (с предупреждением о безопасности)
- Миграция существующих токенов при первом запуске

**Impact:** Критический. Защита учетных данных пользователей.

---

#### 4. Race condition при refresh токенов
**Проблема:** Множественные одновременные запросы вызывали параллельные refresh-запросы.

**Решение:**
- Singleton refresh promise в `/lib/api.ts`
- Все параллельные запросы ждут один общий refresh
- Предотвращение инвалидации токенов из-за race conditions

**Impact:** Критический. Стабильность аутентификации.

---

#### 5. Отсутствие валидации загружаемых файлов
**Проблема:** Нет проверки размера и типа файлов, риск DoS и загрузки вредоносных файлов.

**Решение:**
- Валидация в `/lib/media.ts`:
  - Максимальный размер: 10MB
  - Разрешенные MIME-типы: images (jpeg, png, gif, webp), documents (pdf)
- Проверка перед отправкой на сервер
- Понятные сообщения об ошибках для пользователя

**Impact:** Критический. Защита от DoS и вредоносных файлов.

---

- Graceful handling timeout ошибок
- Понятные сообщения для пользователя

**Impact:** Высокий. Предотвращение зависания UI при проблемах с сетью.

---

#### 7. Утечка серверных ошибок в UI

- User-friendly сообщения на русском языке

**Impact:** Высокий. Безопасность и UX.

---

#### 8. Отсутствие отката при ошибках оптимистичных обновлений
**Решение:**
- Реализован rollback в React Query mutations
- Сохранение предыдущего состояния через `onMutate`
- Восстановление через `onError` с `queryClient.setQueryData()`
- Применено в: статусы пациентов, комментарии, чек-листы

**Impact:** Высокий. Консистентность данных в UI.

## Технический стек исправлений

**Использованные технологии:**
- React Query - для rollback и cache management
- AsyncStorage - для персистентной очереди
- SecureStore - для безопасного хранения токенов
- NetInfo - для мониторинга сети
- DOMPurify - для XSS-защиты
- AbortController - для таймаутов
- TypeScript - для type safety

**Затронутые модули:**
- `/lib/api.ts` - API client с refresh logic и таймаутами
- `/lib/offline-queue.ts` - персистентная очередь мутаций
- `/lib/token-storage.ts` - безопасное хранение токенов
- `/lib/sync.ts` - синхронизация офлайн-данных
- `/lib/media.ts` - валидация файлов
- `/components/**` - UI компоненты с loading/error states
- `/hooks/**` - custom hooks с error handling
