## Расхождение с документацией

### Что написано в API документации

Согласно `/openapi.json`, уведомления должны создаваться автоматически при:

1. **Изменении статуса пациента** → `POST /api/v1/patients/{id}/status`
   - Тип: `STATUS_CHANGE`

2. **Назначении операции** → `POST /api/v1/surgeries`
   - Тип: `SURGERY_SCHEDULED`

3. **Добавлении комментария** → `POST /api/v1/comments`
   - Тип: `NEW_COMMENT`

4. **Истечении срока чек-листа** (автоматически)
   - Тип: `CHECKLIST_EXPIRY`

5. **Напоминании об операции** (автоматически)
   - Тип: `SURGERY_REMINDER`

### Что происходит на самом деле

**Ничего из перечисленного не работает.** Уведомления не создаются автоматически ни при одном из указанных событий.

## Типы уведомлений в API

Согласно схеме `Notification` в OpenAPI:

```typescript
type NotificationType =
  | "STATUS_CHANGE"
  | "NEW_COMMENT"
  | "SURGERY_SCHEDULED"
  | "CHECKLIST_EXPIRY"
  | "SURGERY_REMINDER"
```

## Возможные причины

1. **Функционал не реализован**
   - API документация описывает желаемое поведение, но код не написан

2. **Отключено в конфигурации**
   - Функционал существует, но отключен в настройках

3. **Требуется дополнительная настройка**
   - Нужно настроить воркеры, очереди или триггеры БД

4. **Работает только для определенных ролей**
   - Возможно, уведомления создаются только для определенных типов пользователей
