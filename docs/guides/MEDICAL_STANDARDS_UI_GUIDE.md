# Medical Standards UI - User Guide

## Обзор

Система медицинских стандартов и интеграций теперь доступна в UI приложения. Вы можете добавлять медицинские коды (МКБ-10, SNOMED CT, LOINC) к пациентам и экспортировать данные в ЕМИАС и РИАМС.

## Где найти в приложении

### 1. Экран деталей пациента

Откройте любого пациента из списка. На экране деталей вы увидите:

#### Секция "Медицинские стандарты"

Расположена между "Паспортными данными" и "Действиями". Показывает:

- **Диагнозы (МКБ-10)** - коды заболеваний по международной классификации
- **Процедуры (SNOMED CT)** - коды операций и процедур
- **Наблюдения (LOINC)** - результаты измерений (биометрия глаза)
- **FHIR Resource ID** - идентификатор в системе FHIR (если есть)
- **Статусы интеграций** - синхронизация с ЕМИАС и РИАМС
- **Кнопки экспорта** - отправка данных во внешние системы

Секция сворачивается/разворачивается по клику на заголовок.

#### Кнопка "Медицинские коды"

В разделе "Действия" добавлена новая кнопка для редактирования медицинских метаданных.

## Как добавить медицинские коды

### Шаг 1: Откройте форму редактирования

1. Перейдите на экран деталей пациента
2. Прокрутите до раздела "Действия"
3. Нажмите кнопку **"Медицинские коды"**

### Шаг 2: Добавьте диагноз (МКБ-10)

1. В секции "Диагнозы (МКБ-10)" нажмите на поле выбора
2. Откроется модальное окно с поиском
3. Начните вводить название диагноза (например, "катаракта")
4. Выберите нужный код из списка результатов
5. Код добавится в форму

**Доступные коды катаракты:**
- H25.0 - Старческая начальная катаракта
- H25.1 - Старческая ядерная катаракта
- H25.2 - Старческая катаракта морганиева типа
- H26.1 - Травматическая катаракта
- H26.2 - Осложненная катаракта
- И другие...

### Шаг 3: Добавьте процедуру (SNOMED CT)

1. В секции "Процедуры (SNOMED CT)" нажмите на поле выбора
2. Введите название процедуры (например, "факоэмульсификация")
3. Выберите код из результатов
4. Код добавится в форму

**Доступные процедуры:**
- 172522003 - Факоэмульсификация с имплантацией ИОЛ
- 231744001 - Факоэмульсификация катаракты
- 397544007 - Экстракапсулярная экстракция катаракты
- 308694008 - Имплантация интраокулярной линзы
- И другие...

### Шаг 4: Добавьте наблюдения (LOINC)

1. В секции "Наблюдения (LOINC)" нажмите на поле выбора
2. Введите название измерения (например, "длина оси")
3. Выберите код из результатов
4. Код добавится в форму

**Доступные измерения:**
- 79894-2 - Длина оси правого глаза (mm)
- 79897-5 - Кератометрия K1 правого глаза (D)
- 79898-3 - Кератометрия K2 правого глаза (D)
- 79900-7 - Глубина передней камеры правого глаза (mm)
- И другие...

### Шаг 5: Сохраните изменения

1. Проверьте добавленные коды
2. Нажмите кнопку **"Сохранить"**
3. Форма закроется, коды отобразятся в секции "Медицинские стандарты"

## Как удалить код

В форме редактирования:
1. Найдите код, который хотите удалить
2. Нажмите на иконку **✕** справа от кода
3. Код будет удален из списка

## Экспорт в ЕМИАС

### Что такое ЕМИАС

Единая медицинская информационно-аналитическая система города Москвы.

### Как экспортировать

1. Убедитесь, что у пациента заполнены:
   - ФИО
   - Дата рождения
   - Пол
   - СНИЛС (рекомендуется)
   - Полис ОМС (рекомендуется)

2. В секции "Медицинские стандарты" нажмите кнопку **"Экспортировать в ЕМИАС"**

3. Система проверит данные:
   - Если есть ошибки - покажет список проблем
   - Если все ОК - отправит данные в ЕМИАС

4. После успешного экспорта:
   - Появится статус "Синхронизирован"
   - Отобразится ID пациента в ЕМИАС
   - Кнопка экспорта станет неактивной

### Статусы синхронизации

- **Ожидает** (🕐) - данные готовы к отправке
- **Синхронизирован** (✅) - успешно экспортировано
- **Ошибка** (❌) - не удалось экспортировать

## Экспорт в РИАМС

### Что такое РИАМС

Региональная информационно-аналитическая медицинская система.

### Поддерживаемые регионы

- 77 - Москва
- 50 - Московская область
- 78 - Санкт-Петербург
- 47 - Ленинградская область
- 23 - Краснодарский край
- 61 - Ростовская область
- 66 - Свердловская область
- 74 - Челябинская область
- 54 - Новосибирская область
- 16 - Республика Татарстан

### Как экспортировать

1. Убедитесь, что данные пациента заполнены (как для ЕМИАС)

2. Нажмите кнопку **"Экспортировать в РИАМС"**

3. Выберите регион из списка

4. Система отправит данные в РИАМС выбранного региона

5. После успешного экспорта появится статус синхронизации

## Тестовые данные

### Автоматическое заполнение

Используйте скрипт для добавления тестовых медицинских метаданных:

```bash
npm run seed:medical <EMAIL> <PASSWORD>
```

Скрипт:
- Добавит коды МКБ-10 всем пациентам
- Добавит коды SNOMED CT
- Добавит измерения LOINC
- Создаст статусы синхронизации для некоторых пациентов

### Ручное добавление

Если backend API еще не реализован:
1. Откройте любого пациента
2. Нажмите "Медицинские коды"
3. Добавьте коды вручную через UI
4. Данные сохранятся локально (в console.log)

## Особенности работы

### Поиск кодов

- Поиск работает по коду и описанию
- Регистр не важен
- Поддерживается частичное совпадение
- Результаты обновляются в реальном времени

### Валидация

Перед экспортом система проверяет:
- Обязательные поля (ФИО, дата рождения, пол)
- Формат кодов (МКБ-10, SNOMED, LOINC)
- Доступность региона (для РИАМС)

### Обратная совместимость

- Старые пациенты без медицинских кодов работают как раньше
- Текстовое поле "Диагноз" сохранено как fallback
- Если есть код МКБ-10 - показывается он, иначе текст

## Troubleshooting

### Не отображается секция "Медицинские стандарты"

**Причина:** У пациента нет медицинских метаданных

**Решение:** Добавьте коды через кнопку "Медицинские коды"

### Кнопка экспорта неактивна

**Причина:** Пациент уже синхронизирован

**Решение:** Это нормально, повторный экспорт не требуется

### Ошибка валидации при экспорте

**Причина:** Не заполнены обязательные поля

**Решение:** Проверьте сообщение об ошибке и заполните недостающие данные

### Поиск не находит коды

**Причина:** Используется неправильное название

**Решение:**
- Попробуйте другие варианты (например, "катаракта" вместо "cataract")
- Используйте часть слова
- Проверьте тип кода (ICD-10 для диагнозов, SNOMED для процедур)

### Backend API не реализован

**Причина:** Endpoints еще не созданы на сервере

**Решение:**
- UI работает и показывает mock данные
- Для production требуется реализация backend
- См. документацию: `docs/MEDICAL_STANDARDS.md`
