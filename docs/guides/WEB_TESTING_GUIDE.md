## Тест-кейсы по категориям

### 🔴 Критические исправления (5 тестов)

#### Тест 1: Офлайн-синхронизация
**Цель:** Проверить, что данные не теряются при потере соединения

**Шаги:**
1. Залогиниться в приложение
2. Открыть DevTools → Network → Offline mode
3. Попробовать:
   - Создать комментарий
   - Изменить статус пациента
   - Обновить чек-лист
4. Проверить, что появился индикатор офлайн-режима
5. Включить сеть обратно (Online mode)
6. Подождать 5-10 секунд
7. Обновить страницу

**Ожидаемый результат:**
- ✅ Индикатор офлайн-режима отображается
- ✅ Операции добавлены в очередь (проверить AsyncStorage в DevTools → Application → Local Storage)
- ✅ После восстановления сети данные синхронизировались
- ✅ Изменения видны на сервере

**Проверка в коде:**
- `/lib/offline-queue.ts` - очередь персистентна
- `/lib/sync.ts` - автоматическая синхронизация
- `/hooks/useOfflineSync.ts` - hook управляет состоянием

#### Тест 2: XSS-защита в комментариях
**Цель:** Проверить санитизацию HTML в пользовательском контенте

**Шаги:**
1. Открыть карточку пациента
2. Добавить комментарий с HTML/JS кодом:
   ```
   <script>alert('XSS')</script>
   <img src=x onerror="alert('XSS')">
   <b>Bold text</b>
   ```
3. Сохранить комментарий
4. Обновить страницу

**Ожидаемый результат:**
- ✅ Скрипты НЕ выполняются (нет alert)
- ✅ Опасные теги удалены или экранированы
- ✅ Безопасные теги (b, i, strong) могут работать (зависит от конфигурации DOMPurify)
- ✅ Текст отображается корректно

**Проверка в коде:**
- `/components/patient/CommentThread.tsx` - использует DOMPurify на web
- Web: санитизация через DOMPurify
- Mobile: React Native Text (автоматически безопасен)

#### Тест 3: Безопасное хранение токенов
**Цель:** Проверить, что токены хранятся в localStorage (web) и защищены

**Шаги:**
1. Залогиниться в приложение
2. Открыть DevTools → Application → Local Storage
3. Найти ключи: `auth_token`, `refresh_token`
4. Проверить значения (должны быть JWT токены)
5. Разлогиниться
6. Проверить, что токены удалены

**Ожидаемый результат:**
- ✅ Токены хранятся в localStorage (web) или SecureStore (mobile)
- ✅ Токены - валидные JWT (можно проверить на jwt.io)
- ✅ После logout токены удаляются
- ✅ При повторном входе токены обновляются

**Проверка в коде:**
- `/lib/token-storage.ts` - платформо-специфичное хранение
- Web: localStorage (с предупреждением в документации)
- iOS/Android: SecureStore (аппаратное шифрование)

#### Тест 4: Race condition при refresh токенов
**Цель:** Проверить, что множественные запросы не вызывают параллельные refresh

**Шаги:**
1. Залогиниться в приложение
2. Открыть DevTools → Network
3. Сделать токен невалидным (изменить в localStorage)
4. Быстро открыть несколько вкладок/запросов:
   - Список пациентов
   - Уведомления
   - Профиль
5. Наблюдать за Network запросами

**Ожидаемый результат:**
- ✅ Только ОДИН запрос на `/auth/refresh`
- ✅ Все остальные запросы ждут завершения refresh
- ✅ После refresh все запросы повторяются с новым токеном
- ✅ Нет ошибок 401 после refresh

**Проверка в коде:**
- `/lib/api.ts` - singleton refresh promise
- Все параллельные запросы используют один refresh

#### Тест 5: Валидация загружаемых файлов
**Цель:** Проверить ограничения на размер и тип файлов

**Шаги:**
1. Открыть карточку пациента → Медиафайлы
2. Попробовать загрузить:
   - Файл > 10MB (должен отклониться)
   - .exe файл (должен отклониться)
   - .jpg файл < 10MB (должен загрузиться)
   - .pdf файл < 10MB (должен загрузиться)
3. Проверить сообщения об ошибках

**Ожидаемый результат:**
- ✅ Файлы > 10MB отклоняются с понятным сообщением
- ✅ Неразрешенные типы (exe, zip, etc.) отклоняются
- ✅ Разрешенные типы (jpg, png, pdf) загружаются
- ✅ Сообщения об ошибках на русском языке

**Проверка в коде:**
- `/lib/media.ts` - валидация перед отправкой
- Максимальный размер: 10MB
- Разрешенные MIME: image/*, application/pdf

**Экраны для проверки:**

- Чек-листы
- Уведомления
- Медиафайлы

**Ожидаемый результат:**
- ✅ Все экраны имеют loading state
- ✅ Все экраны имеют error state с retry
- ✅ Все экраны имеют empty state с подсказкой

## Автоматизация (будущее)

Для автоматизации этих тестов можно использовать:
- **Playwright** - E2E тесты
- **Cypress** - альтернатива Playwright
- **Jest + React Testing Library** - unit/integration тесты
- **Detox** - для React Native mobile

**Пример Playwright теста:**
```python
from playwright.sync_api import sync_playwright

with sync_playwright() as p:
    browser = p.chromium.launch(headless=True)
    page = browser.new_page()
    page.goto('http://localhost:8081')
    page.wait_for_load_state('networkidle')

    # Тест логина
    page.fill('input[type="email"]', 'admin@gmail.com')
    page.fill('input[type="password"]', '123123123')
    page.click('button[type="submit"]')
    page.wait_for_url('**/patients')

    # Проверка списка пациентов
    assert page.locator('.patient-card').count() > 0

    browser.close()
```
